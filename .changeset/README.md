# Changesets

Hello and welcome! This folder has been automatically generated by `@changesets/cli`, a build tool that works
with multi-package repos, or single-package repos to help you version and publish your code.

## What is a changeset?

A changeset is a piece of information about changes made in a branch or commit. It holds three bits of information:

1. What packages should be bumped and what kind of bump it should be (major, minor, or patch)
2. A summary of the changes (written in markdown)
3. Who made the changes

When changesets are merged to `main`, the Release workflow consumes them, bumps package versions, and updates CHANGELOGs.

## Adding a changeset

Run `bun changeset` from the repo root and follow the prompts before opening a PR into `next`.
